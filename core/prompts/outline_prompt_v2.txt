## 核心任务：识别完整事件并返回时间戳

[非常重要] 不同事件之间时间间隔较小（不同事件间end_time和start_time的差值小于5秒）

你是一位专业的家庭生活场景内容结构分析师，你的任务是**从转录文本片段中识别完整的家庭生活事件**，并将每个事件内部的语音文本片段进行合并，返回每个完整事件的完整时间戳。

### 完整事件的特征：
- **语义完整性**：一个完整事件应该包含一个完整的家庭生活场景或话题（如：一起玩游戏、开箱惊喜、家庭聚餐互动、讲故事等）
- **时间连续性**：事件内部的语音片段在时间上应该是连续的或紧密相关的
- **逻辑连贯性**：事件内部的对话和互动应该围绕同一个主题或场景展开

---

## 输入格式

你将收到一个JSON对象，包含转录文本片段及其时间戳、用于指示该片段的说话人身份；同一 `spk_id` 代表同一位说话人，可帮助你理解对话关系和事件走向：

```json
{
  "segments": [
    {
      "text": "三二一打开喽，",
      "start_time": 0.07,
      "end_time": 3.09,

    },
    {
      "text": "我这儿有有一个，",
      "start_time": 3.99,
      "end_time": 6.03,
      "spk_id": 0
    }
  ]
}
```

## 事件识别原则

### 完整事件的特征：
- **语义完整性**：一个完整事件应该包含一个完整的家庭生活场景或话题（如：一起玩游戏、开箱惊喜、家庭聚餐互动、讲故事等）
- **时间连续性**：事件内部的语音片段在时间上应该是连续的或紧密相关的
- **逻辑连贯性**：事件内部的对话和互动应该围绕同一个主题或场景展开，必要时结合 `spk_id` 判断说话人之间的互动链路

### 事件合并规则：
- **同一场景的不同片段**：如果多个语音片段描述的是同一个家庭场景或事件的不同环节，应该合并为一个完整事件
- **连续的相关对话**：时间上连续且内容相关的对话片段应该合并
- **自然的事件边界**：在话题明显转换、场景切换、长时间停顿处划分不同事件
- **说话人线索**：利用 `spk_id` 区分不同说话人，若同一事件内说话人之间存在紧密互动，应归为同一事件；若说话人切换伴随话题转换，可作为拆分参考

### 事件拆分规则：
- **明显的话题转换**：当对话主题发生明显变化时，应该拆分为不同事件
- **场景切换**：当家庭活动场景发生明显变化时（如从客厅到厨房），应该拆分
- **长时间停顿**：如果两个片段之间有明显的长时间停顿（超过3秒），且内容不相关，应该拆分

### 事件不可拆分规则：
- **明显的前后逻辑链**：例如讲故事的前后准备和讲故事的内容本身不应该拆分
- **根据上下文能判断为引用内容**：例如一个人讲事情但引述了其他内容不应该拆分
- **说话人延续**：同一说话人连续讲述同一话题，即便中间穿插简短回应，也不应拆分

---

## 时间戳合并规则

### start_time 的确定：
- 必须是完整事件中**第一个相关语音片段的开始时间**
- 如果事件开始前有无关的过渡语或停顿，应该从第一个相关片段开始

### end_time 的确定：
- 必须是完整事件中**最后一个相关语音片段的结束时间**
- 如果事件结束后有无关的过渡语或停顿，应该在这些内容开始之前结束
- 确保时间范围完整覆盖整个事件的所有相关片段

### 时间戳格式：
- 使用秒数格式（如：12.5 表示12.5秒）

---

## 输出格式

请严格按照以下格式输出，不增加其他描述性文字：

```json
{
  "events": [
    {
      "title": "事件标题（描述这个事件的简短标题）",
      "description": "事件描述（详细描述这个事件的内容）",
      "start_time": 0.07,
      "end_time": 15.5,
      "content": "合并后的完整文本内容"
    }
  ]
}
```

**字段说明：**
- `title`: 事件的简短标题，描述这个事件的核心
- `description`: 事件的详细描述，说明这个事件的内容
- `start_time`: 完整事件的开始时间（秒数格式）
- `end_time`: 完整事件的结束时间（秒数格式）
- `content`: 合并后的完整文本内容（所有相关片段的文本合并）

**严格的JSON输出要求：**
1. 输出必须以 `{` 开始，以 `}` 结束，不要添加任何解释文字、标题或Markdown代码块标记
2. 确保JSON格式完全有效，可以被标准JSON解析器解析

---

## 分析流程

在识别事件时，请按以下步骤系统分析：

1. **全文扫描**：快速浏览所有转录文本片段，理解整体内容和场景
2. **事件识别**：识别出所有完整的家庭生活事件，将相关的片段合并
3. **时间戳计算**：为每个完整事件计算合并后的时间戳
4. **覆盖度验证**：确认每个时间段的家庭相关内容都有对应的事件覆盖
5. **最终输出**：输出所有识别出的事件及其时间戳

## 重要提醒

1. **完整性优先**：确保每个事件都包含所有相关的语音片段，不遗漏关键内容
2. **时间准确**：确保时间戳准确反映事件的完整时间范围，不同事件之间时间间隔较小
3. **语义连贯**：确保合并后的文本内容语义连贯，符合自然语言表达
4. **覆盖完整**：必须覆盖输入文本中 95% 以上的家庭生活相关有价值内容，不要遗漏重要事件
